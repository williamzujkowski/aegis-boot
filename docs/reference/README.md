## Adding a new auto-generated doc

1. Write the generator as a feature-gated bin target in `crates/aegis-cli/src/bin/` (pattern established by `constants_docgen.rs` and `cli_docgen.rs`).
2. Emit the doc under `docs/reference/NAME.md` or `docs/reference/schemas/NAME.json`.
3. Add a drift-check job to `.github/workflows/ci.yml` that runs the generator in `--check` mode.
4. Add a row to the index table above.

This follows the "single source of truth" pattern established by [#286](https://github.com/aegis-boot/aegis-boot/issues/286): docs derive from code, not copy from it.
